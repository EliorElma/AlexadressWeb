# אלכסה בריידל — קוד האתר

אתר פשוט: HTML+CSS+JS רגילים, בלי build step, בלי תלות בשום פריימוורק.
אפשר לפתוח את `index.html` ישירות בדפדפן כדי לראות אותו.

## מבנה הקבצים
```
website-export/
  index.html        כל התוכן והמבנה של העמוד
  css/style.css      כל העיצוב (צבעים, פונטים, רספונסיביות)
  js/main.js         תפריט מובייל + לוגיקת מבחן הסטייל + שליחת וואטסאפ
  images/            כל התמונות (עכשיו placeholders — להחליף בתמונות אמיתיות)
```

## איך להחליף תמונות אמיתיות
כל שם קובץ ב-`images/` מתאר בדיוק מה צריך להיות שם. גררי תמונה חדשה
לתיקייה עם **אותו שם קובץ בדיוק** (או עדכני את הנתיב ב-`index.html`):
- `hero.jpg` — תמונת השמלה הראשית בעמוד הבית
- `dress-1.jpg` עד `dress-6.jpg` — צד קדמי של 6 השמלות בקולקציה (גם מופיעות ברצועות התמונות הנעות)
- `dress-1-back.jpg` עד `dress-6-back.jpg` — צד אחורי של אותן שמלות (מוצג כשלוחצים על כפתור ה-Front/Back שעל כל תמונה בקולקציה)
- `studio.jpg` — תמונת הסטודיו
- `bride-1.jpg` עד `bride-6.jpg` — כלות אמיתיות
- `mood-1.jpg`, `mood-2.jpg`, `mood-4.jpg`, `mood-5.jpg` — רגעים מהעולם של אלכסה
- `ig-1.jpg` עד `ig-6.jpg` — פוסטים מאינסטגרם
- `logo.png` — הלוגו (כבר הלוגו האמיתי שלך)

לוידאו: יש 2 סלוטים מסומנים בהערה בתוך `index.html` (חיפשי `video-slot`).
תחליפי את ה-`<div class="tile video-slot">` בתגית `<video>` עם הקובץ שלך,
לפי הדוגמה שבהערה.

## איך לערוך את התוכן — קובץ אחד: `js/site-config.js` ⭐
כל התוכן של האתר יושב באובייקט `SITE` בקובץ `js/site-config.js` (מבנה JSON):
- `head_pic` — תמונת הפתיחה
- `first_band` — רשימת התמונות החתוכות ברצועה העליונה (כמה שרוצים)
- `dresses` — השמלות: שם, מחיר, קישור, front / back / detail
  (**detail אופציונלי** — מוחקים את השורה ואין כפתור Detail לשמלה)
- `quiz` — השאלות, התשובות, ולאיזו שמלה כל תשובה מובילה (`answerDressMap`)
- `second_band` — התמונות ברצועה של THE DRESS THAT YOU
- `real_brides` — כלות אמיתיות: שם, תאריך, תמונה
- `moments` — תמונות הקולאז' (כמה שרוצים)
- `testimonials` — המלצות: טקסט + חתימה
- `instagram_pics` — תמונות האינסטגרם
- `whatsappNumber`, `web3formsKey`, `instagramHandle` — הגדרות כלליות

מוסיפים/מורידים פריט = מוסיפים/מוחקים שורות ב-config, שומרים ודוחפים לגיט.
הרצועות הנעות מסתדרות לבד לכל כמות תמונות.

טקסטים קבועים (כותרות, USP וכו') עדיין ב-`index.html`.

## איך להעלות לגיטהאב
1. הורידי את התיקייה `website-export` ופתחי אותה בטרמינל.
2. אם עדיין אין לך ריפו:
   ```
   git init
   git add .
   git commit -m "אתר אלכסה בריידל"
   ```
3. צרי ריפו חדש וריק ב-GitHub (github.com → New repository).
4. חברי וטעני:
   ```
   git remote add origin https://github.com/<username>/<repo-name>.git
   git branch -M main
   git push -u origin main
   ```

## איך לפרסם אונליין (בחינם) + לחבר דומיין
**האפשרות הכי פשוטה: Netlify**
1. היכנסי ל-netlify.com, התחברי עם GitHub.
2. "Add new site" → "Import an existing project" → בחרי את הריפו.
3. Build command: השאירי ריק. Publish directory: `.` (שורש) — האתר סטטי, אין build.
4. לחצי Deploy — תוך שניות יהיה לך לינק חי (`your-site.netlify.app`).
5. לחיבור דומיין משלך: Site settings → Domain management → Add custom domain,
   ותעדכני את רשומות ה-DNS אצל ספק הדומיין (Namecheap / GoDaddy / כל מי שקנית
   אצלו) לפי ההוראות ש-Netlify מציגה (בדרך כלל רשומת CNAME).

**חלופות שעובדות אותו דבר** (גם הן חינמיות לאתר סטטי כזה): Vercel,
GitHub Pages (Settings → Pages → Deploy from branch), Cloudflare Pages.

## לידים במייל (Web3Forms) — צעד אחרון קטן
מבחן הסטייל כבר מחובר לשליחת ליד במייל **בנוסף** לוואטסאפ. נשאר רק להדביק מפתח:
1. היכנסי ל-https://web3forms.com והזיני את כתובת המייל שאליה יגיעו הלידים.
2. הם יראו/ישלחו לך **Access Key** — העתיקי אותו.
3. פתחי את `js/main.js`, בראש הקובץ ב-`CONFIG`, והחליפי את הערך של
   `web3formsKey: 'YOUR_WEB3FORMS_ACCESS_KEY'` במפתח שקיבלת.
4. שמרי ודחפי לגיט — מעכשיו כל מי שמשלימה את המבחן שולחת לך מייל עם השם,
   הטלפון והשמלות שהתאימו לה (וגם נפתח לה וואטסאפ כמו קודם).

עד שתדביקי מפתח — האתר ממשיך לעבוד רגיל ופותח וואטסאפ בלבד.
